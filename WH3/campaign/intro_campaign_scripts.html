Campaign Camera Marker

During campaign tutorial gameplay it is common to have scripted sections where the player has to inspect camera positions in order to progress. Camera markers are placed down in set positions, and when the player moves the camera close to them a cutscene plays which delivers narrative regarding the marker. The intention behind this is that the player learns about camera controls whilst narrative content is delivered to them.

An intro_campaign_camera_marker represents an individual camera marker in a scripted section such as this. One or more markers are created and associated with a intro_campaign_camera_positions_advice object in order to make such a sequence work.

Example

The following script gives a simple example of the creation of multiple intro_campaign_camera_marker objects, linked to a single intro_campaign_camera_positions_advice object.

Example:

-- get a handle to the enemy settlement and its display position
local enemy_settlement = cm:get_region(cm:get_faction(ENEMY_FACTION_KEY):home_region():name()):settlement()
local enemy_settlement_x = enemy_settlement:display_position_x()
local enemy_settlement_y = enemy_settlement:display_position_y()

-- create enemy settlement marker
local marker_enemy_settlement = intro_campaign_camera_marker:new(
    "enemy_settlement",
    enemy_settlement_x,
    enemy_settlement_y,
    12
)

-- add actions to enemy settlement marker cutscene
marker_enemy_settlement:action(
    function()
        cm:scroll_camera_from_current(false, 6, {enemy_settlement_x, enemy_settlement_y, 14.1, 0, 4.0}) end, 0)
    end,
    0
)
marker_enemy_settlement:action(function() cm:show_advice("intro_enemy_settlement_advice_01") end, 1)
marker_enemy_settlement:action(function() cm:scroll_camera_from_current(true, 5, {174.3, 51.4, 26.1, 0, 14.0}) end, 7)

-- get a handle to the player's army and its diplay position
local players_army = cm:get_character_by_cqi(PLAYER_ARMY_CQI)
local players_army_x = players_army:display_position_x()
local players_army_y = players_army:display_position_y()

-- create player army marker
local marker_player_army = intro_campaign_camera_marker:new(
    "player_army",
    players_army_x,
    players_army_y,
    10
)

-- add actions to playe army marker cutscene
marker_player_army:action(
    function()
        cm:scroll_camera_from_current(false, 5, {players_army_x, players_army_y, 14.1, 0, 4.0})
    end,
    0
)
marker_player_army:action(function() cm:show_advice("intro_player_army_advice_01") end, 1)
marker_player_army:action(function() cm:scroll_camera_from_current(true, 4, {174.3, 51.4, 26.1, 0, 12.0}) end, 6)

-- create intro_campaign_camera_positions_advice object, associated with markers above
local ca = intro_campaign_camera_positions_advice:new(
    "intro_camera_movement_objective",
    function() progress_from_intro_campaign_camera_advice() end,
    {marker_enemy_settlement, marker_player_army}                -- table containing all markers
)

-- start behaviour
ca:start()        

Campaign Select And Attack Advice

During campaign tutorial gameplay it is common to have scripted sections where the player has to select an army and then attack a target in order to progress. An intro_campaign_select_and_attack_advice facilitates this behaviour - locking the UI, showing advice, compelling the player to move their army and controlling the movement of the army as this is done.

Back to top

Creation

Intro:new(number player cqi, number enemy cqi, string initial advice, string sleection objective, table markers)
Creates and returns an intro_campaign_select_and_attack_advice object.

Parameters:

1

number

Command queue index of the player's army that they have to attack with.

2

number

Command queue index of the target enemy army.

3

string

Initial advice key, from the advice_threads table. This is delivered when the objective is issued to the player, and should set up the task narratively.

4

string

Initial selection objective key, from the scripted_objectives table. This is delivered when the objective is issued to the player, and should set up the task narratively.

5

table

a table containing one or more intro_campaign_camera_marker objects to be associated with this camera positions advice object.

Returns:

  1. intro_campaign_camera_positions_advice camera positions advice
